# Contributing to CityLife360-PowerBI-Dashboard

Thanks for your interest in contributing! Below is a simple workflow and checklist.

How to contribute
1. Fork the repository.
2. Create a feature branch:
   - git checkout -b feature/short-description
3. Make changes and commit with clear messages.
4. Push to your fork and open a Pull Request against the repository's main branch.
5. In your PR description, include:
   - What you changed and why.
   - Screenshots for visual/UI changes.
   - Any migration steps or PBIX/data changes needed.

PR checklist
- [ ] Code / measure names use consistent naming conventions.
- [ ] PBIX changes include notes on data model or Date table changes.
- [ ] No production-sensitive data is committed.
- [ ] Visual changes include before/after screenshots.

Contact
If you have questions, open an issue or contact the repo owner: https://github.com/Vikas-Giri
